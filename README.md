# Drowsiness / Yawn Detector

Real-time yawning detection for up to 6 faces via webcam. MediaPipe FaceMesh extracts 468 landmarks → 1404 features per face, fed into a custom logistic regression classifier.

## Repository structure

```
├── demo.py                        # Real-time webcam detector (entry point)
├── logreg.py                      # Custom logistic regression (NumPy)
├── mypipe.py                      # Pipeline: normalize → PCA → model
├── final_model.pkl                # Trained model (pickle)
│
├── modules/                       # Training-side modules (used by notebooks)
│   ├── dfe.py                     # DrowsinessFeatureExtraction (MediaPipe wrapper)
│   ├── logreg.py                  # LogRegModel (training variant)
│   ├── pipe.py                    # MyPipeline (training variant)
│   └── pca.py                     # DDDPCA helper
│
├── notebooks/
│   ├── 1_feature_extraction.ipynb # Extract landmarks from image dataset → CSV
│   └── 2_model_training.ipynb     # Grid search CV → best hyperparameters
│
├── data/
│   ├── raw/                       # ← place datasets here (see Dataset Setup)
│   │   ├── YawDD/                 # Training set
│   │   └── ISDDS/                 # Test set
│   ├── facial_drowsiness_data.csv # Generated by notebook 1
│   └── isdds_data.csv             # Generated by notebook 1
│
├── logs/
│   └── log.txt                    # Images skipped during feature extraction
│
├── requirements.txt
└── install.sh
```

## Quick start

### 1. Install

```bash
git clone <repo-url> && cd <repo>
chmod +x install.sh && ./install.sh
source venv/bin/activate
```

### 2. Run the real-time detector

```bash
python demo.py          # press ESC to quit
```

> Requires `final_model.pkl` in the repo root. If you don't have it, run the notebooks below to produce one.

### 3. Train from scratch (optional)

**Dataset setup** — place images under `data/raw/`:
```
data/raw/YawDD/yawn/      ← yawning images
data/raw/YawDD/no_yawn/   ← neutral images
data/raw/ISDDS/yawn/
data/raw/ISDDS/no_yawn/
```

Then open the notebooks in order:
```bash
jupyter notebook notebooks/
```

| Notebook | What it does |
|---|---|
| `1_feature_extraction.ipynb` | Reads images → extracts landmarks → saves `data/*.csv` |
| `2_model_training.ipynb` | Loads CSVs → 10-fold grid search CV → prints best hyperparameters |

## Requirements

| Requirement | Detail |
|---|---|
| Python 3.9–3.12 | mediapipe doesn't support 3.13+ |
| Webcam | USB or built-in; `/dev/video0` on Linux |
| `final_model.pkl` | In repo root for `demo.py` |

## Platform notes

**macOS** — grant Terminal camera access: System Settings → Privacy → Camera.

**Ubuntu/Debian** — if the detector window won't open:
```bash
sudo apt-get install -y libgl1 libglib2.0-0
```

**WSL2** — webcam passthrough not supported natively; use [usbipd-win](https://github.com/dorssel/usbipd-win) or run on native Windows.

**Windows (native)** — replace `source venv/bin/activate` with `venv\Scripts\activate`.

## How the detector works

```
Webcam frame
  → MediaPipe FaceMesh  (468 landmarks × xyz = 1404 features/face, up to 6 faces)
  → MyPipeline          (normalize disabled at inference; optional PCA)
  → LogisticRegression  (custom NumPy implementation)
  → Overlay             "TERDETEKSI MENGUAP" (yawning, red) | "NETRAL" (blue)
```

Normalization is off at inference (`pipe.normalize = False`) — MediaPipe coordinates are already in a consistent scale. Iris landmarks (469–478) are stripped; only base 468 used.

## Troubleshooting

| Symptom | Fix |
|---|---|
| `ModuleNotFoundError: mediapipe` | Run `install.sh` or activate venv first |
| Webcam not found | Try `cv2.VideoCapture(1)` for external cams |
| `pickle` load error | `logreg.py` and `mypipe.py` must be in the same directory as `demo.py` |
| Low FPS | Reduce `max_num_faces` in `demo.py` line 24, or remove per-landmark dot drawing |
| Window won't open (Linux) | `sudo apt-get install libgl1` |
